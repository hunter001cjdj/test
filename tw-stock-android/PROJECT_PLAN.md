# 台股分析與優質股票篩選 Android App 工作計畫

## 專案目標
- [ ] 建立一款 Android 手機軟體，協助使用者分析台股。
- [ ] 提供基本面、技術面、成長潛力與風險面向的整合分析。
- [ ] 篩選出相對優質、未來可能具成長機會的台股標的。
- [ ] 支援即時版資料分析與即時市場變化判讀。
- [ ] 依照風險 / 報酬組合分類股票。
- [ ] 提供 1 到 5 顆星推薦指數與推薦 / 不推薦原因。
- [ ] 以「輔助決策」為定位，避免包裝成保證獲利或投資建議。

## 專案定位
- [ ] 目標使用者：想研究台股、但需要系統化整理資訊的投資人。
- [ ] 核心價值：快速整理資料、建立評分機制、提供觀察清單。
- [ ] 產品形式：Android App。
- [ ] 第一版目標：先完成 MVP，不追求完整券商級功能。

## 產品功能規劃

### 1. 股票資料查詢
- [ ] 支援搜尋台股代號與公司名稱。
- [ ] 顯示個股基本資訊：產業、股價、成交量、市值。
- [ ] 顯示歷史價格走勢。
- [ ] 顯示財務數據摘要。
- [ ] 顯示即時或近即時行情變化。
- [ ] 顯示即時漲跌、成交量變化與波動狀態。

### 2. 優質股票篩選
- [ ] 設計基本面篩選條件：營收成長、EPS、ROE、毛利率、負債比。
- [ ] 設計技術面條件：均線趨勢、量價變化、相對強弱。
- [ ] 設計風險條件：波動度、籌碼集中度、獲利穩定性。
- [ ] 產出「候選優質股票清單」。
- [ ] 建立風險 / 報酬四象限分類。

### 3. 成長潛力分析
- [ ] 建立成長評估指標：營收年增、獲利趨勢、產業成長性。
- [ ] 建立簡易評分模型。
- [ ] 顯示「可能成長」的理由與風險提示。
- [ ] 區分短期題材與中長期成長型標的。

### 4. 推薦指數與理由說明
- [ ] 建立 1 到 5 顆星推薦指數。
- [ ] 顯示推薦原因。
- [ ] 顯示不推薦原因。
- [ ] 顯示風險提示與觀察重點。

### 5. 觀察清單與提醒
- [ ] 讓使用者收藏關注個股。
- [ ] 顯示自選股觀察清單。
- [ ] 預留價格提醒與條件提醒功能。

### 6. 報表與解讀
- [ ] 用簡單文字解釋分析結果。
- [ ] 提供分數卡或雷達圖視覺化。
- [ ] 顯示「為什麼被列入優質候選」。
- [ ] 顯示「有哪些潛在風險」。
- [ ] 顯示所屬風險 / 報酬分類。
- [ ] 顯示推薦星等與綜合結論。

## 風險 / 報酬分類規則
- [ ] 高風險高報酬
說明：波動大、題材強、成長空間大，但回撤風險高。
- [ ] 低風險高報酬
說明：基本面穩定、成長趨勢清楚、風險相對可控，是優先關注型標的。
- [ ] 高風險低報酬
說明：風險偏高但上行空間不足，通常應列為低優先或避免。
- [ ] 低風險低報酬
說明：穩定但爆發力有限，適合保守觀察，不一定適合追求成長型使用者。

## 推薦指數規格
- [ ] 5 顆星：高優先推薦，條件完整，基本面與成長性強，風險可接受。
- [ ] 4 顆星：偏推薦，整體不錯，但仍有部分風險或資料待觀察。
- [ ] 3 顆星：中立觀察，條件普通，沒有明顯優勢也沒有明顯致命缺點。
- [ ] 2 顆星：偏不推薦，風險高於潛在回報，或條件不足。
- [ ] 1 顆星：不推薦，風險偏大且成長或報酬預期不佳。

## 推薦 / 不推薦原因輸出格式
- [ ] 推薦原因至少列出 2 到 4 點。
- [ ] 不推薦原因至少列出 2 到 4 點。
- [ ] 每檔股票需產出明確結論，不只顯示分數。
- [ ] 原因來源需對應到實際指標，例如營收成長、ROE、負債比、波動度、量價趨勢。
- [ ] 避免空泛描述，例如「感覺不錯」或「可能會漲」。

# 四象限分類與推薦模型

## 1. 模型目標

將股票轉換成：
- 風險 / 報酬四象限分類
- 1 到 5 星推薦指數
- 推薦原因
- 不推薦原因

## 2. 核心分數

### 2.1 風險分數 Risk Score

分數範圍：`0 ~ 100`

可參考指標：
- 近 20 日波動度
- 最大回撤
- 負債比
- 籌碼集中風險
- 獲利穩定度

分數越高代表風險越高。

### 2.2 報酬 / 成長分數 Reward Score

分數範圍：`0 ~ 100`

可參考指標：
- 營收年增率
- EPS 成長率
- ROE
- 毛利率趨勢
- 價格趨勢強度
- 產業成長性

分數越高代表未來成長或報酬潛力越高。

## 3. 四象限分類規則

### 高風險高報酬

- Risk Score >= 60
- Reward Score >= 60

適合：
- 願意承受高波動的人

### 低風險高報酬

- Risk Score < 60
- Reward Score >= 60

適合：
- 優先關注的理想型標的

### 高風險低報酬

- Risk Score >= 60
- Reward Score < 60

適合：
- 通常列入低優先或避免

### 低風險低報酬

- Risk Score < 60
- Reward Score < 60

適合：
- 保守觀察，但成長爆發力有限

## 4. 推薦星等公式

建議先使用簡化版本：

```text
Final Score = (Reward Score * 0.5) + (基本面分數 * 0.2) + (技術面分數 * 0.15) + (穩定度分數 * 0.15) - (Risk Score * 0.25)
```

再映射到星等：

- `>= 75`：5 星
- `>= 60`：4 星
- `>= 45`：3 星
- `>= 30`：2 星
- `< 30`：1 星

## 5. 推薦原因產生規則

若以下條件成立，可列為推薦原因：

- 營收年增率高於門檻
- EPS 持續成長
- ROE 高於標準
- 毛利率穩定或上升
- 股價站上中長期均線
- 成交量支持趨勢
- 所屬產業具成長性

推薦原因範例：

- 近四季營收與 EPS 持續成長
- ROE 表現優於同產業平均
- 技術面維持多頭趨勢
- 所屬產業中長期需求明確

## 6. 不推薦原因產生規則

若以下條件成立，可列為不推薦原因：

- 波動過大
- 負債比過高
- 獲利不穩
- 短線漲多且量價背離
- 產業景氣不明
- 成長性不足

不推薦原因範例：

- 近期波動度過高，短線風險偏大
- 負債比偏高，財務壓力較重
- 獲利成長不穩定
- 目前上行空間與承擔風險不成比例

```json
{
  "stockId": "2330",

  "riskScore": 38,
  "rewardScore": 82,
  "quadrant": "低風險高報酬",
  "starRating": 5,
  "recommendReasons": [
    "營收與 EPS 維持成長",
    "ROE 表現穩定",
    "產業成長趨勢明確"
  ],
  "avoidReasons": [
    "短線漲幅過大時需留意追價風險"
  ],
  "summary": "屬於低風險高報酬類型，適合列入優先研究清單。"
}
```

## 8. 後續優化方向

- 改成可調權重模型
- 加入產業別差異化評分
- 加入機器學習或回測驗證
- 加入風格偏好，例如穩健型 / 成長型 / 短線型
